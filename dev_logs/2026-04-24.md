# 2026-04-24 研发日志｜Plotly 虚拟支气管镜：相机漫游、双面观感与交互稳定性

## 摘要（TL;DR）

本日工作的核心目标是把 Plotly 的“沿路径移动”的相机漫游，推进到更接近 VTK/PyVista 的“内镜观感”，并将其工程化为 **单 HTML 内可交互、可复现、稳定流畅** 的产物。阶段性成果包括：

- **离线相机提示量**：在 Python 侧预计算 `cam.fwd/lookDist`（射线锥可视性），注入 HTML，前端优先使用，显著减少弯道“看墙/看偏”。
- **Plotly 相机跳变修复**：冻结 data→scene 映射基准 `navCamBasis`，消除某些帧 axis range/basis 重算导致的“视角突然跳到外部”。
- **内镜观感增强**：引入 headlight、材质重配、几何平滑、双层填充层，缓解凹面发黑；并在 UI 层面实现 **腔外/腔内/融合** 三态（近似“双面材质”）。
- **交互范式沉淀**：用 `opacity≈0` 隐藏替代 `visible` 切换，显著降低卡顿并保持 look-around；播放中任何手动操作统一暂停，避免竞争。
- **面板融合与弱同步**：切片浏览器嵌入导航面板；导航帧 Z(mm) 与切片采用最近邻同步（非一一对应），提升定位效率且不引入跳变。
- **运行时换色**：用户可选择气管配色主题（深/浅成对），并持久化到 `localStorage`。

**推荐演示产物（当前最佳）**：

## 2. 方法：从“切向看前方”到“路径负责走、射线负责看”

### 2.1 问题陈述（为什么要引入射线求交）

仅用折线切向/前方若干点确定 `forward`，在窄弯与偏心截面常出现：

- **看墙/看偏**：视线指向壁面或偏离“最大开口”方向；
- **注视点不合理**：`center` 落在壁后或距离过近，产生压迫感；
- **折角抖动**：折线顶点切向跳变，平滑与贴合难两全。

VTK/PyVista 的优势之一是能回答：**从当前 eye 沿某方向看过去，多久会撞壁？** 该“自由距离”是直接的几何信号，适合用于选择 `fwd` 与 `lookDist`。

### 2.2 离线相机提示量（注入 HTML）

对路径点 \(p_i\) 预计算：

- `cam.fwd[i]`：锥形多射线可视性方向（并与路径切向混合，确保推进一致性）
- `cam.lookDist[i]`：依据正前方自由距离估计，并 clamp 到合理范围（避免“盯太远”穿帮）

前端相机更新原则：

- `eye = pts[i]`
- `center = eye + cam.fwd[i] * cam.lookDist[i]`（优先使用离线提示，缺失则回退旧逻辑）

## 3. Plotly 内镜渲染与交互：关键工程经验

### 3.1 “黑面/偏暗”的成因与对应策略

- **成因判断**：更像 Plotly `mesh3d` 的光照模型与光源位置导致背光面过暗（而非相机跑出腔体）。
- **策略组合**：
  - headlight：相机更新时动态绑定 `lightposition`；
  - `vertexnormalsepsilon=0`、`facenormalsepsilon=0`：止血法降低阴影伪影；
  - 材质重配：降低高光闪烁，增加可读性；
  - 几何平滑：从法线/三角形质量层面减少黑块；
  - 双层填充：极低透明度纯 ambient 的“填充层”抬亮凹面。

### 3.2 近似“双面材质”：腔外/腔内/融合三态

Plotly `Mesh3d` 无原生 two-sided lighting 开关。工程近似做法是：

- **外壁层**：用于腔外参照（`气管区域(3D充气法)`）
- **翻面内壁层**：用于腔内观察（`气管内壁(腔内)`）
- **融合模式**：外壁 + 翻面内壁同时显示，实现近似“双面材质”的观感

### 3.3 交互稳定性：为什么 `opacity≈0` 比 `visible=false` 更稳

- `scene.camera` 使用“场景域坐标”，必须依赖稳定的 data→scene 映射；本项目用 `navCamBasis` 冻结该映射，避免某帧重算造成跳变。
- 对 3D trace 改 `visible` 往往触发 scene bounds / axis range 重算，导致卡顿或交互异常（主观表现为 look-around 失效或视角突变）。
- 因此切换模式优先采用：**保持 `visible=true`，用 `opacity≈0`（如 0.001）实现近似隐藏**，以维持场景边界稳定。

### 3.4 播放与用户操作的竞争：统一 `stop()` 的原因

播放依赖定时器持续 `relayout(scene.camera)`，会持续抢相机控制权。若用户此时切片/换色/切模式/拖路径帧，容易出现竞争导致卡顿或状态不同步。因此将 `stop()` 作为所有手动交互的前置动作，能把系统从“自动控制”切回“用户控制”。

## 4. UI：面板融合、弱同步与运行时可调参数

### 4.1 面板融合

切片浏览器改为可嵌入块，运行时挂载到导航面板内（减少多浮窗抢占视野与焦点）。

### 4.2 导航→切片弱同步（最近邻）

导航帧与切片抽样（`section-interval`）不是一一对应。采用“路径点 Z(mm) → 最近切片”同步：

- 足以提供空间定位；
- 不强绑定采样体系，减少跳变；
- 同步触发时禁用滚动（`__NAV_SYNCING__`），避免打断相机交互。

### 4.3 运行时换色（深/浅成对）

提供多套主题（绿/蓝/红/米/肉），并对外壁/内壁/填充层联动上色；使用 `localStorage` 持久化偏好。

## 7. 可量化指标（建议后续纳入每次 run 记录）

- **前方自由距离**：`s_front` 分布（均值/5%分位），过小代表频繁怼墙
- **贴壁风险**：环绕射线 `min(s_k)`，过小代表 eye 靠壁
- **视线抖动**：相邻帧 `angle(fwd[i], fwd[i-1])` 分布
- **路径曲率**：离散外接圆半径最小值（越小越急拐）
