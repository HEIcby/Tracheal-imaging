## 目标 1｜第 3 章 3.2 “气道区域提取”实验设计（用于 3.2.6）

### 1.1 研究问题（论文可写）

- **Q1**：相比“只做阈值 + 形态学 + 逐切片/简单连通域”的传统做法，本文的气道区域提取（以 3D 分析/充气法为核心）是否能**更稳健地抑制体外空气与孔洞干扰**，得到更完整的气道区域？
- **Q2**：在不同样本（`dicom1/2/3`）与不同参数扰动下（阈值、形态学、Z 裁剪、种子），本文方法是否更**鲁棒**？

### 1.2 对照组与控制变量（实验设计写死）

> 原则：一次实验只动 1 个因素；其余全部冻结在 `run_args.json`，并在日志中写明。

- **Method-A（本文方法 / Proposed）**：`--use-3d-analysis`（3D 分析/充气法）+（可选）`--expand-by-shell` 扩展策略。
- **Method-B（传统/弱基线 1）**：不启用 3D 分析（去掉 `--use-3d-analysis`），仅用阈值 + 形态学 +（若代码默认还有其它步骤，需在 `run_args.json` 明确）。
- **Method-C（消融 / 弱基线 2，可选）**：启用 3D 分析但**去掉扩展**（若扩展可开关），用于说明“扩展策略”的贡献。

**冻结项（所有组必须一致）**：
- 数据集：`dicom{1|2|3}`
- Z 裁剪：`--z-min/--z-max`
- 种子：`--start-z`（或 `--start-idx`）
- 二值化策略：`--fixed-threshold t`（优先固定阈值；必要时再加 1 组 `t±Δ`）
- 形态学：`--closing-iters c --erosion-iters e`
- 重建参数：`--size/--iso/--step`（用于保证几何一致，避免混入网格质量因素）

### 1.3 评价指标（不依赖人工标注也能落地）

> 现实约束：当前未必有逐体素 GT 标签；因此优先采用“可复现、可自动计算、能反映气道提取质量”的代理指标。若后续补 GT，可把 Dice/IoU 作为追加指标。

- **完整性（Connectivity/Continuity）**：
  - **气道主连通域体素占比**：\(r_{cc}=\frac{|CC_{max}|}{|mask|}\)。越接近 1 越好（越少碎片/漏连）。
  - **中心线是否可生成 + 路径点数 \(N\)**：可作为“是否可用于导航”的硬指标（失败即不可用）。
- **抗体外空气泄漏（Leakage）**：
  - **mask bounding box 体积利用率**：\(r_{bbox}=\frac{|mask|}{|bbox(mask)|}\)。过低常意味着大量稀疏泄漏/飞点（需结合可视化判读）。
  - **离散小连通域数量**：\(K\)（去掉小体素阈值后计数）。越少越好。
- **几何合理性（用于后续导航的可用性）**：
  - 若能产出导航路径：复用已有 `_navigation_metrics`：\(L\)、\(R_{min}\)、`d_min/d_p05`（这里作为“下游任务可用性”指标）。

**产物（每个组至少保留）**：
- HTML：展示阈值候选、最终气道区域（绿色层）、中心线/导航线（若启用）。
- `run_args.json`：冻结项全记录。
- `run_metrics.json`：新增字段建议：
  - `_airway_extraction_metrics`: `{ r_cc, r_bbox, num_components, mask_voxels, bbox_volume_voxels, ... }`

### 1.4 实验矩阵（最小可写入论文的一页表）

### 1.5 预期结论（论文可写）

## 目标 2｜第 5 章 5.3.1 “射线求交法”降低抖动：叙述 + 实验设计

### 2.1 方法叙述（写入论文的“机制解释”草案）

> 问题：仅用路径切向（或“看向前方若干点”）生成相机 forward，在弯道、偏心截面与局部狭窄处，forward 容易在相邻帧发生跳变：一方面导致视线撞墙（中心落在壁面附近），另一方面会引起视频视野抖动（方向/注视点快速抖动）。
>
> 解决：本文引入**射线求交法**，在每个路径点处沿候选方向发射多条射线，与气道内壁网格求交，得到“前方自由距离”与更稳定的可视方向；再将该方向与路径推进方向混合，并用求得的自由距离确定 lookDist。这样可把“看哪里”从纯局部切向几何，提升为由全局可见空间约束的稳定决策，从而显著降低弯道处的视线抖动与撞墙频率。

### 2.2 对照组（A/B）与固定项

- **Cam-A（Baseline）**：不使用射线求交提示量，forward/lookDist 仅由路径切向/前视点规则给出（即“旧逻辑”）。
- **Cam-B（Proposed）**：使用射线求交预计算的 `cam.fwd[i]` 与 `cam.lookDist[i]`（即“新逻辑/提示量注入”）。

**固定项**：
- 同一条路径点序列（同一 `--navigation-line` 输出；同一 `N` 与采样策略）
- 同一 mesh（同一 run 的气道网格）
- 同一播放速度/采样帧率（便于对比）

### 2.3 可量化指标（“抖动”落到数字）

> 目标：把“观感抖动”转为可重复计算的数值指标，作为 5.3.1 的实验支撑。

- **角速度抖动（方向变化）**：
  - 记每帧 forward 单位向量 \(f_i\)，定义 \(\Delta\theta_i=\arccos(\mathrm{clip}(f_i\cdot f_{i-1},-1,1))\)。
  - 指标：`mean(Δθ)`、`p95(Δθ)`、`max(Δθ)`；越小越稳。
- **注视点抖动（center 变化）**：
  - 记 \(c_i = eye_i + f_i \cdot lookDist_i\)。
  - 指标：`mean(|c_i-c_{i-1}|)`、`p95(|c_i-c_{i-1}|)`；越小越稳。
- **撞墙/贴壁风险（可视自由距离）**：
  - 若射线求交可提供“正前方自由距离” \(d^{free}_i\)（或用距离变换近似），统计：
    - `wall_hit_rate = count(d_free < τ)/N`（例如 \(τ=2\)mm 或 5mm）
    - `p05(d_free)`；越大越安全。

**产物建议**：
- 每次相机对照导出 `run_camera_metrics.json`（或写入 `run_metrics.json` 的 `_camera_metrics`），按 `cam_algo` 分组存放：
  - `cam_algo`: `baseline_tangent` / `ray_intersection`
  - `stats`: `{ delta_theta_mean, delta_theta_p95, center_step_p95, wall_hit_rate, ... }`

### 2.4 实验矩阵（最小可写入论文的一张表）

- **C1（主对照）**：选 1 个代表性数据集（优先 `dicom1` 或 `dicom2`），同一路径上 Cam-A vs Cam-B 对比上述指标，并截取弯道处视频帧作为定性证据。
- **C2（泛化，选做）**：在 `dicom1/2/3` 上各做一次 Cam-A vs Cam-B（3 组即可形成“跨样本一致”结论）。

### 2.5 预期结论（论文可写）

- Cam-B（射线求交）应在弯道处显著降低 `p95(Δθ)` 与 `wall_hit_rate`，同时减少“看墙/看偏”的主观频次；定量与定性一致时，可作为 5.3.1 “相机轨迹生成方法优越性”的核心证据。

  - **抖动指标（Δθ/center_step）**在三例中未呈现“稳定下降”的一致趋势：不建议在 5.3.1 写成“显著降低抖动”的强结论；建议将优势重点落在“可见空间约束提升视野合理性/安全裕度”，并在后续补“弯道/狭窄段分段统计”强化抖动论证。
