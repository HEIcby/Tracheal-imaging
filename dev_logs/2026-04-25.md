### 4) 导航线指标工程化（今日必须补齐的“表格化输出”）

对每次带导航线的 run，至少记录：

- **路径几何**：
  - 路径点数 \(N\)、路径长度 \(L\)（mm）
  - 最小转弯半径估计 \(R_{min}\)（mm）
  - 平滑度：相邻帧方向变化（主观也可，若能量化更好）
- **贴壁风险（建议作为核心指标之一）**：
  - 取路径点到气腔边界的距离（可用距离变换近似）：`d_min / d_p05 / d_med`（mm）
  - 解释：`d_min` 太小意味着相机/导管路径贴壁风险高

> 注：如果今天暂时来不及把贴壁风险量化落代码，也要先把“主观贴壁/怼墙频率”作为临时指标写入 Run 记录，保证 A/B 可比。

### 5) 路径优化 + 传统算法对比（今天至少做 1 轮可对比实验）

**对比的最低要求**：在同一份 `dicom3` 输出（同一 Z 裁剪、同一阈值/形态学）下，产出至少两条路径并对比指标：

- **Baseline**：`ALG=nav_current`（现有 `--navigation-line` 输出）
- **Optimized**：在不改主流程的前提下做“后处理版路径”之一：
  - `ALG=nav_smooth`：对路径做平滑 + 等弧长重采样（减少抖动）
  - `ALG=nav_push_center`：用距离变换把路径点往“腔中心”推回（减少贴壁）

**传统路径算法候选（择 1~2 个）**：

- `ALG=dijkstra_dt`：在 3D 气腔体素图上建图，代价与距离变换相关（越靠中心代价越低），做最短路（Dijkstra/A*）
- `ALG=dt_ridge`：距离变换脊线/局部最大（更贴近“中轴线”的经典思路），再做曲线拟合/平滑
- `ALG=centerline_spline`：把当前中心线/骨架线做 B-spline 平滑 + 曲率约束（作为“传统平滑中心线”基线）

**对比输出**（写入日志即可）：
- `R_min` 是否提升（更大更“插管友好”）
- 贴壁指标是否改善（`d_min / d_p05` 变大）
- 播放观感：怼墙/看偏频率是否下降

### 6) 播放卡顿/帧率提升（做“可复现”的前端实验）

以导航面板的 **步进间隔(ms)** 作为最小可控旋钮（当前可选区间 **10–100ms**），在同一 HTML 上做对比：

- `FPS=ms100`（相对保守/更稳定）
- `FPS=ms60`（常用）
- `FPS=ms40`（更激进）
- `FPS=ms20 / ms10`（极限：用于观察是否仍卡）

每次记录：
- 是否明显更流畅
- 是否出现“相机失控/掉帧严重/浏览器假死”
- 期间是否触发 trace 切换（建议保持 opacity 隐藏法，不用 visible 切换）

若 `ms` 降到 40/20 仍卡：
- 优先怀疑：**每帧 relayout 成本过大** 或 **路径点数过多**。
  - 用面板 **点数降采样(步长)** 做对比：`ds=1/2/3/4/6`（步长越大点数越少，通常越流畅）


## 导航线指标工程化（目标 B）落地内容

### 已工程化并自动输出的指标

- **路径长度 \(L\)**：`path_length_mm`（mm）
  - 含义：导航线折线的总长度（相邻点欧氏距离求和）
- **最小转弯半径 \(R_{min}\)**：`min_turn_radius_mm_estimated`（mm）
  - 含义：三点外接圆半径的最小值（越大越平滑、更插管友好）
- **贴壁距离统计（mm）**：`wall_dist_mm`
  - `d_min`：最贴壁点的离壁距离（极端风险点）
  - `d_p05`：5% 分位离壁距离（更稳健的贴壁风险指标，适合 A/B）
  - `d_med`：中位数离壁距离（反映整体居中程度）
  - `d_mean`：平均离壁距离（受极端点影响更大）

### 输出位置（确保可复现/可对比）

- **终端**：在 `✓ 导航线已添加到3D图: ...点` 后追加一行 `导航指标: L=... | Rmin=... | dmin=... | ...`
- **HTML 底部实验信息**：新增独立板块 **“工程化参数统计”**（表格：指标/中文解释/数值）
- **参数 JSON 归档**：在实验信息的参数 JSON 中新增字段 `_navigation_metrics`（保留原始数值，便于后处理/批量统计）
